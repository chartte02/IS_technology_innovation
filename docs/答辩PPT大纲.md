- 抓包: Scapy
- GUI: PyQt5
- 匹配算法: Aho-Corasick 自动机

## 第五部分：总结与展望 (3-4页)

### P17. 项目难点与解决方案
| 难点 | 解决方案 |
|------|----------|
| TCP 流重组 | 四元组 + LRU 淘汰 |
| 多模式匹配效率 | AC 自动机 O(n) |
| 误报率控制 | 白名单 + 上下文感知 + 基线学习 |
| 加密流量检测 | JA3 指纹 + TLS 代理（扩展） |
| 高吞吐性能 | 生产者-消费者 + 端口索引 |

### P18. 创新点
- 混合检测架构（误用+异常互补）
- AC 自动机 + 正则两阶段匹配
- TCP 流重组抗逃避
- 灵活的 YAML 特征库
- 支持基线学习自适应

### P19. 不足与改进方向
- 当前不足:
  - 不支持 IPv6
  - 不支持加密流量检测
  - 特征库不够全面
- 改进方向:
  - 集成 JA3/JA4 TLS 指纹
  - 增加机器学习模型（Isolation Forest / AutoEncoder）
  - 实现攻击链关联分析
  - 导出 Snort/Suricata 兼容规则

### P20. 总结
- 完成了完整的 IDS 系统设计与实现
- 覆盖了 7 大类 250+ 种攻击检测
- 双引擎架构互补，兼具准确性和覆盖面
- 可扩展的模块化设计
