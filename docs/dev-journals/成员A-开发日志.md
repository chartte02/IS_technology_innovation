### 我解决的 3 个最有价值的问题

1. **AC 自动机 key 复用 bug**：`id(key)` 在 Python 中可能被 GC 回收后复用，导致多规则映射到同一 key → 改为直接使用 tuple 作 dict key，从根本上消除了这个隐患
2. **误报率过高问题**：正常路径 `/select-course` 会触发 SQL 注入告警 → 实现四层降噪（白名单 IP → URL 路径过滤 → Referer 降级 → Cookie 降级），正常路径 0 误报
3. **跨成员接口不一致**：alert 格式缺少 `type`/`description` 字段 → 对照 CLAUDE.md §5.2 逐一补全，确保与 `AlertManager` 无缝对接

### 我的 3 个关键技术贡献

1. **91 条攻击特征规则体系**：从最初 58 条扩展到 91 条，新增 SSRF/XXE/SSTI/NoSQL/蚁剑/冰蝎/哥斯拉等 18 条自定义规则 + 15 条 Suricata 导入规则，覆盖 9 大类攻击
2. **Suricata 规则导入器**：实现行业标准规则格式解析器，支持 8 种关键字、classtype 自动映射、`$HTTP_PORTS` 变量，可导入社区 3000+ 条规则
3. **上下文感知误报降噪系统**：四层过滤机制（白名单→URL路径→Referer→Cookie），配合成员C 的基线降噪，共同完成 PDF 要求的误报自动降噪功能

### 我从开源项目中学到的 3 点

1. **ThreatWire 的 AC 自动机 + 多索引架构**：将规则按协议/端口分类建立快速索引，非标准端口场景下可达到 10.5x 加速比
2. **Suricata 规则的 classtype 体系**：通过 classtype 自动映射 category + severity，避免了手工分类的繁琐和错误
3. **OWASP Top 10 的攻击分类覆盖检查**：对照 OWASP 2021 确认我们的规则库覆盖了注入/XXE/SSRF/SSTI/反序列化等所有关键类别
