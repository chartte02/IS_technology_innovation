### 2. 遇到的问题与解决

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| AC 自动机中 `id(key)` 作字典 key 不可靠 | Python 的 `id()` 在对象回收后可被复用，导致 key 冲突 | 改用 tuple 直接作为 dict key（`self._ac_sig_map[key] = sig`），已由队友修复 |
| AC 模式未统一转小写 | 大小写不敏感匹配时，AC 自动机存入原始大小写的 pattern，无法匹配小写载荷 | `ac_pattern = pattern.lower()` 后再加入自动机，已由队友修复 |
| 测试文件 `test_signature_match.py` 被意外删除 | git 操作中丢失 | `git restore` 恢复 |
| Scapy 提示 "No libpcap provider available" | Windows 未安装 Npcap | 安装 Npcap（WinPcap 兼容模式），不影响测试运行 |
