### 2. 遇到的问题与解决

| 问题 | 现象 | 原因 | 解决方案 |
|------|------|------|----------|
| **测试初期动态阈值不生效** | 端口扫描自适应测试始终返回 `threshold=50`（固定值） | `check_all()` 每轮只产生 1 个观察值，无法达到 `min_samples=5` | 改为多轮 `update → check_all` 循环，积累 ≥5 个观察值后再触发异常 |

## 答辩素材汇总（Day 7 填写）

### 我解决的 3 个最有价值的问题

1. **`_check_baseline_deviation` set 与 float 类型比较崩溃** — Python 短路求值在特定条件下返回了空 set，导致与浮点数比较时报类型错误。这使基线偏离检测完全不可用。

2. **`_check_login_pattern` 缺少 SSH `failed password` 模式** — 最常见的 SSH 登录失败日志格式未被覆盖，导致约 80% 的 SSH 暴力破解无法被检测到。

3. **ML 模型特征维度不匹配** — Isolation Forest 训练时过滤了零方差特征列（8→6 维），但 predict 时仍传 8 维向量，导致 `X has 8 features, but model expects 6` 报错。通过保存 `_valid_cols` 掩码并在 predict 时同样过滤解决。

### 我的 3 个关键技术贡献

1. **自动化测试套件** — `tests/test_anomaly_detection.py`，15 个测试用例一键验证，为后续拓展提供回归保障。

2. **ML 异常检测模块** — `core/ml_anomaly.py`，基于 Isolation Forest 的无监督学习，8 维特征向量，支持自动训练/推理/模型持久化，与 AnomalyDetector 无缝集成。

3. **攻击链关联 + 误报降噪 + 动态阈值自适应** — 独立模块 `core/attack_chain.py`（7 阶段攻击链串联）、`core/alert_filter.py`（三层降噪：白名单+资产重要性+基线偏离）和 `AdaptiveThresholdManager`（μ±kσ 动态阈值，5 种检测器自适应），答辩时可直接展示关联前后的对比数据和自适应效果。

### 我从开源项目中学到的 3 点

1. **IF+RF 两阶段方案**（参考 R6）：Isolation Forest 做初筛 + Random Forest 做二次分类，可以将误报率降低 88.6%。

2. **多层时间窗口设计**（参考 R5 Slips）：用 5 分钟 / 1 小时 / 1 天三层滑动窗口比单一 60 秒窗口更能捕捉不同时间尺度的异常。

3. **特征工程的关键性**（参考 R9）：特征向量的维度选择直接影响检测效果，8 维特征需要覆盖协议、流量、时间三个维度。
