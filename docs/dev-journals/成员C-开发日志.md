## 答辩素材汇总（Day 7 填写）

### 我解决的 3 个最有价值的问题

1. **`_check_baseline_deviation` set 与 float 类型比较崩溃** — Python 短路求值在特定条件下返回了空 set，导致与浮点数比较时报类型错误。这使基线偏离检测完全不可用。

2. **`_check_login_pattern` 缺少 SSH `failed password` 模式** — 最常见的 SSH 登录失败日志格式未被覆盖，导致约 80% 的 SSH 暴力破解无法被检测到。

3. **零测试覆盖** — 接手时异常检测模块没有任何自动化测试。编写了 11 个测试用例覆盖全部 7 种检测器及白名单跳过、正常流量不误报、基线持久化等边界场景。

### 我的 3 个关键技术贡献

1. **自动化测试套件** — `tests/test_anomaly_detection.py`，11 个测试用例一键验证，为后续拓展提供回归保障。

2. **阈值参数调优** — 对 4 项检测阈值进行了系统性的检出率/误报率对比实验，找到了最佳平衡点，灵敏度平均提升 2 倍。

3. **真实流量验证** — 用 CIC-IDS-2017 数据集 50000 包验证了异常检测在真实环境下的有效性。

### 我从开源项目中学到的 3 点

1. **IF+RF 两阶段方案**（参考 R6）：Isolation Forest 做初筛 + Random Forest 做二次分类，可以将误报率降低 88.6%。

2. **多层时间窗口设计**（参考 R5 Slips）：用 5 分钟 / 1 小时 / 1 天三层滑动窗口比单一 60 秒窗口更能捕捉不同时间尺度的异常。

3. **特征工程的关键性**（参考 R9）：特征向量的维度选择直接影响检测效果，8 维特征需要覆盖协议、流量、时间三个维度。
