# 开发日志目录

> 每位成员记录自己的开发过程，最终整理为答辩素材。
> 请在每天开发结束后花 5-10 分钟填写。

## 为什么需要记录

| 目的 | 说明 |
|------|------|
| **答辩素材** | 日志中的细节（遇到的问题、解决思路、性能对比）直接用于 PPT |
| **个人贡献证明** | 记录每一项改动，答辩时能说清楚"我做了什么" |
| **经验沉淀** | 踩过的坑写下来，其他人可以避免 |
| **AI Agent 协作记录** | 记录 Agent 帮助了哪些工作、效率提升多少 |

## 日志模板说明

每个成员的模板分为 7 个必填板块：

1. **每日进度** — 今天完成了什么
2. **遇到的问题与解决** — 踩了什么坑、怎么解决的（答辩核心素材）
3. **Agent 协作记录** — 哪些地方用了 AI Agent、效果如何（答辩亮点）
4. **技术决策** — 为什么选这个方案而不是那个
5. **性能/测试数据** — 跑了什么测试、结果是什么（答辩用数字说话）
6. **参考项目借鉴** — 从开源项目中借鉴了什么
7. **明日计划** — 明天要做什么

## 如何使用 Agent 填写日志

```
"读取 docs/dev-journals/成员A-开发日志.md，
帮我在今天的日期下补充以下内容：
- 今天修复了 misuse_detector.py 的 AC 自动机索引 bug
- 遇到的问题：pyahocorasick 在 Windows 上编译失败
- 解决方案：改用 SimpleMultiPatternMatcher fallback
- Agent 协助：让 Claude 帮我写了 fallback 实现"

Agent 会帮你整理格式，你只需要口述做了什么。
```
